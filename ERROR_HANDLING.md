# Error Handling & Fallback Mechanisms\n\n## Architecture Overview\n\nThe Angel One integration includes multiple layers of error handling:\n\n```\nFrontend (angel_one.js)\n    ↓\nAPI Calls (fetchJSONWithFallback)\n    ↓\nBackend (app.py endpoints)\n    ↓\nAngel One SmartAPI (smartapi-python)\n    ↓\nMock Data (Fallback)\n```\n\n---\n\n## Layer 1: Frontend Error Handling\n\n### angel_one.js Error Catching\n\n```javascript\n// All API calls wrapped in try-catch\nasync function fetchAngelOneQuotes(symbols = 'NIFTY,SENSEX,TCS,RELIANCE,INFY') {\n    try {\n        const response = await fetch(...);\n        const data = await response.json();\n        \n        if (data.quotes) {\n            angelOneData.quotes = data.quotes;\n            updateAngelOneQuotesUI();\n            return data.quotes;\n        }\n    } catch (error) {\n        console.error('Error fetching Angel One quotes:', error);\n        // UI remains responsive, shows last known data\n    }\n}\n```\n\n**What happens when API fails:**\n- Error logged to console (helps debugging)\n- UI doesn't crash\n- Shows \"loading\" state to user\n- Retries on next refresh interval (5 seconds)\n\n---\n\n## Layer 2: Backend API Error Handling\n\n### app.py Endpoint Protection\n\n```python\n@app.route('/angel-status', methods=['GET'])\ndef get_angel_status():\n    try:\n        # Create or reuse Angel One connection\n        angel_api = AngelOneAPI(\n            client_id=os.getenv('ANGEL_ONE_CLIENT_ID'),\n            api_key=os.getenv('ANGEL_ONE_API_KEY'),\n            password=os.getenv('ANGEL_ONE_PASSWORD'),\n            totp_secret=os.getenv('ANGEL_ONE_TOTP')\n        )\n        \n        authenticated = angel_api.authenticate()\n        \n        return jsonify({\n            'status': 'connected' if authenticated else 'disconnected',\n            'authenticated': authenticated,\n            'message': 'Connected to Angel One SmartAPI' if authenticated else 'Connection failed'\n        })\n    \n    except Exception as e:\n        # Graceful error response\n        return jsonify({\n            'status': 'error',\n            'authenticated': False,\n            'message': str(e),\n            'error': 'Failed to connect to Angel One'\n        }), 500\n```\n\n---\n\n## Layer 3: Angel One Module Fallback\n\n### angel_one_api.py Fallback Mechanism\n\n```python\nclass AngelOneAPI:\n    def get_quote(self, symbol):\n        try:\n            # Try real Angel One API\n            quote = self.smartapi.getQuote(...)\n            return quote\n        except Exception as e:\n            # Fallback to mock data\n            print(f\"Failed to get real quote: {e}\")\n            return self._get_mock_quote(symbol)\n    \n    def _get_mock_quote(self, symbol):\n        # Returns synthetic data when API fails\n        return {\n            'ltp': 19500.00,\n            'open': 19400.00,\n            'high': 19550.00,\n            'low': 19350.00,\n            'close': 19500.00\n        }\n```\n\n---\n\n## Error Scenarios & Responses\n\n### Scenario 1: Missing Credentials\n\n**Problem:** `.env` file not created or credentials missing\n\n**Frontend Response:**\n```\n🔴 Angel One: Disconnected\n\"Configuration required\"\n```\n\n**Backend Response:**\n```json\n{\n    \"status\": \"error\",\n    \"authenticated\": false,\n    \"error\": \"Missing ANGEL_ONE_CLIENT_ID in environment\"\n}\n```\n\n**Solution:** User sees status indicator and must configure `.env`\n\n---\n\n### Scenario 2: Invalid Credentials\n\n**Problem:** Credentials in `.env` are incorrect\n\n**Frontend Response:**\n```\n🔴 Angel One: Disconnected\nShows last known data or mock data\n```\n\n**Backend Response:**\n```json\n{\n    \"status\": \"error\",\n    \"authenticated\": false,\n    \"error\": \"Invalid credentials provided\"\n}\n```\n\n**Solution:** Automatic retry every 30 seconds\n\n---\n\n### Scenario 3: TOTP Expired/Invalid\n\n**Problem:** TOTP code expired or wrong\n\n**Backend Error:**\n```\nAuthenticationError: Invalid OTP\n```\n\n**Frontend Fallback:**\n- Uses mock data\n- Shows yellow warning status\n- Retries authentication\n\n---\n\n### Scenario 4: Network Disconnected\n\n**Problem:** No internet connection\n\n**Frontend Error:**\n```javascript\nTypeError: fetch failed\nFetch request blocked\n```\n\n**Handling:**\n- Caught in `fetchJSONWithFallback`\n- Shows \"Connecting...\" status\n- Retries when connection restored\n\n---\n\n### Scenario 5: Backend Server Down\n\n**Problem:** `python app.py` not running\n\n**Frontend Error:**\n```\nFailed to fetch from http://127.0.0.1:5000\nERR_CONNECTION_REFUSED\n```\n\n**Handling:**\n- Connection refused caught\n- Shows red \"Disconnected\" status\n- Displays helpful message in console\n\n---\n\n### Scenario 6: API Rate Limit Exceeded\n\n**Problem:** Too many requests to Angel One API\n\n**Backend Response:**\n```json\n{\n    \"status\": \"error\",\n    \"error\": \"Rate limit exceeded. Try again later.\"\n}\n```\n\n**Frontend Handling:**\n- Increases refresh interval (backs off)\n- Uses last known data\n- Retries with exponential backoff\n\n---\n\n## Monitoring & Debugging\n\n### credentials | \"Configure Angel One credentials in .env\" | Create .env file |\n| Invalid credentials | \"Disconnected - check credentials\" | Verify credentials |\n| Network error | \"Unable to connect - check internet\" | Check connection |\n| Rate limit | \"Too many requests - please wait\" | Wait 1 minute |\n| API error | \"Angel One service temporarily unavailable\" | Retry |\n\n---\n\n## Recovery Mechanisms\n\n### Automatic Retry Logic\n\n```javascript\n// Retry every 5 seconds for quotes\nsetInterval(() => {\n    if (angelOneData.authenticated) {\n        fetchAngelOneQuotes();\n    }\n}, ANGEL_ONE_CONFIG.refreshInterval); // 5 seconds\n\n// Check status every 30 seconds\nsetInterval(() => {\n    checkAngelOneStatus();\n}, ANGEL_ONE_CONFIG.statusCheckInterval); // 30 seconds\n```\n\n**Benefits:**\n- Automatic recovery from temporary failures\n- No user intervention needed\n- Data updates as soon as connection restored\n\n---\n\n## Testing Error Scenarios\n\n### Test 1: Simulate Missing Credentials\n```powershell\n# Remove or comment out lines in .env\n# Start backend\n# Observe: Red \"Disconnected\" **Log errors to console** for debugging\n3. **Show user-friendly messages** (not stack traces)\n4. **Provide fallback data** when APIs fail\n5. **Implement automatic retry** with backoff\n6. **Monitor error rates** in production\n7. **Test all error scenarios** before deployment\n\n---\n\n## Production Readiness Checklist\n\n- ✅ All API calls wrapped in try-catch\n- ✅ Fallback to mock data when
